Create `.github/workflows/ci.yml` with the following structure:

1. **Trigger:** `on: push: branches: [main]` + `pull_request: branches: [main]` (per D-01)
2. **Job: `ci`** with strategy matrix `os: [ubuntu-latest, macos-latest, windows-latest]` (per D-02) and `fail-fast: false` (per D-10)
3. **Steps (in order):**
   - `actions/checkout@v4` (per D-04)
   - `dtolnay/rust-toolchain@stable` with `components: rustfmt, clippy` (per D-04)
   - `Swatinem/rust-cache@v2` (per D-04)
   - `cargo fmt --check` (per D-03)
   - `cargo clippy -- -D warnings` (per D-03)
   - `cargo test` (per D-03)

4. **Platform-specific considerations (per D-15, D-17):**
   - The `cargo test` step should use `--no-default-features` on non-Windows OR add `--exclude` for crates that depend on `windows-dpapi` if needed. Since `windows-dpapi` is gated by `cfg(windows)` in Cargo.toml, it should compile fine — but tests using it will fail on Linux/macOS.
   - For `headless_chrome` tests: these need Chrome installed. On ubuntu, add a step to install Chrome or skip those tests with `--skip browser` if test names contain "browser". On macOS, Chrome is usually available. On Windows, it depends.
   - **Pragmatic approach:** Use `cargo test` without filtering first. If `headless_chrome` tests fail in CI due to missing Chrome, add a conditional test filter.

5. **YAML structure:**
```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    name: ${{ matrix.os }}
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          components: rustfmt, clippy
      - uses: Swatinem/rust-cache@v2
      - name: fmt
        run: cargo fmt --check
      - name: clippy
        run: cargo clippy -- -D warnings
      - name: test
        run: cargo test
```
